## Gotchas (cost real time if forgotten)

- **Windows exe file-lock:** if `cargo build` link fails `Access is denied. (os error 5)`, the user has `infinity_orchestrator.exe` open. The *compile* succeeded; just tell them to close it and retry the link. Not a code error.
- **Don't trust `grep "^Finished"` chained** ‚Äî `cargo test --lib | tail -1` catches a trailing blank line; use `grep "test result:"`. Background-build tail can mislead; read the output file.
- **Symbol-glyph coverage (cmap-verified, see HANDOFF caveat):** Poppins TTFs are a 217-glyph Latin subset. `FiraCodeNerdFont-Light.ttf` (full, 10,801 glyphs) HAS `‚àû ‚úì ‚Üê ‚Üí` (base-FiraCode) but NOT `‚öÝ`/`‚öô`/`‚ò∞` (Misc-Symbols block, U+2600‚Äì26FF). Render base-FiraCode glyphs in `firacode_nerd`; **paint vectors** for Misc-Symbols/emoji (precedent: `left_rail.rs` nav icons, `destination_not_empty.rs::paint_warning_triangle`). Don't assume coverage ‚Äî check the cmap (`python -m pip install --user fonttools`; `TTFont(p).getBestCmap()`).
- **Stable rustfmt skips let-chain files** (`if let ‚Ä¶ && ‚Ä¶`, e.g. `orchestrator_app.rs`) atomically ‚Äî they're left unchanged, not corrupted. Expected.
- **`cargo clippy --fix` / `cargo fmt --all` are forbidden** ‚Äî they rewrite the whole crate incl. protected BIO. Always scope to changed/redesign files.
- **Clippy policy is settled:** default `cargo clippy -- -D warnings` is the real gate (redesign passes). The user's strict pedantic/nursery profile is NOT a blocking gate (BIO half is un-passable by construction). Redesign code was triaged clean (fix Cat-1 / justified module-scoped `#![allow]` Cat-2 colour-casts & Cat-3 churn). Don't re-open this.
- **`derive(Default)` traps:** `ThemeChoice` defaults to `Dark` (NOT first variant `Light`); `RedesignSettings` is a manual `impl Default` (`validate_paths_on_startup: true`) ‚Äî must NOT be derived. The serde-default tests assert these; if they pass, value-identity held.
